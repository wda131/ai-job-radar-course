# AI 求职雷达课程汇报说明

## 项目简介（组队表约 50 字）

面向高校求职者的智能求职管理系统，提供职位检索、能力画像、可解释匹配、收藏投递、进度管理、模拟面试与异步消息提醒，帮助学生完成求职全流程管理。

## 技术路线与课程对应

1. Vue 3、Vue Router、Axios、Vite 和 Node.js 完成前端单页应用。
2. Spring Boot、Spring MVC、Controller–Service–Mapper 分层和 MyBatis Plus 对应课程 SSM/Boot 路线。
3. Spring Cloud Alibaba、Nacos、Gateway、OpenFeign 对应 day03 微服务拆分。
4. Redis 缓存职位列表和详情，RabbitMQ 异步传递投递状态事件。
5. MySQL 8 保存用户、职位、匹配、收藏、投递、面试和通知数据。

## 六人分工（按功能模块）

每位成员负责一个完整业务模块，从 Vue 页面、接口、Service、Mapper 到数据库和测试，符合“模块数量不少于人数”的要求。仓库未提供真实姓名，答辩前将下表“成员一至成员六”替换为实际姓名。

| 成员 | 业务模块 | 前后端工作与答辩内容 |
| --- | --- | --- |
| 成员一（队长） | 用户与个人档案 | 登录鉴权、用户画像、user-service、JWT、系统联调与进度管理 |
| 成员二 | 职位雷达 | 检索筛选、job-service、分页查询、Redis 缓存与命中演示 |
| 成员三 | 智能匹配 | 匹配报告、match-service、OpenFeign、四维评分规则 |
| 成员四 | 收藏与投递 | 收藏/投递看板、application-service、状态流转、RabbitMQ 发布 |
| 成员五 | 模拟面试 | 四题训练、interview-service、作答评分、反馈和面试记录 |
| 成员六 | 消息中心 | 通知页面、notification-service、RabbitMQ 消费、幂等与已读管理 |

## 演示脚本（8–10 分钟）

1. 介绍求职痛点、功能边界和课程技术栈。
2. 展示 Nacos 中 Gateway 与六个业务服务。
3. 登录并完善档案，在职位雷达筛选岗位；重复查询展示 Redis key。
4. 收藏并投递岗位，打开消息中心展示 RabbitMQ 异步通知。
5. 生成匹配报告，解释技能、城市、薪资和经验四维评分。
6. 创建模拟面试，展示四道题、作答、即时评分和历史记录。
7. 更新投递状态，再次展示消息通知；最后展示测试结果和 Git 提交记录。

## 稳妥边界

保留六个核心业务模块，不增加自动投递、即时聊天、Elasticsearch 或 Docker。可选技术不影响评分主线，现场只依赖课程明确要求的 MySQL、Nacos、Redis 和 RabbitMQ。
