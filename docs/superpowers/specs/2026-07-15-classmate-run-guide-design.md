# 同学交接运行指南设计

## 目标

让第一次接触项目的同学仅依赖 GitHub 仓库说明，在 Windows 电脑上完成环境准备、数据库初始化、IDEA 启动、前端启动、功能验证，以及可选的 BOSS Chrome 扩展安装和真实岗位导入。

## 读者与成功标准

- 读者没有使用过开发者本机的绝对路径和环境变量。
- 主体项目不安装 Chrome 扩展也能启动；扩展仅用于导入真实 BOSS 岗位。
- 推荐路线使用 Docker Desktop 启动 MySQL、Redis、Nacos、RabbitMQ 和 Elasticsearch，Java 微服务仍在 IDEA 中启动。
- 同时保留无 Docker 的原生安装路线，并明确需要设置的环境变量和数据库初始化命令。
- 每个阶段都有端口或命令验证点，失败时可定位到具体组件。

## 文档结构

1. README 增加“发给同学：从零运行”入口和最短流程。
2. `docs/CLASSMATE_RUN_GUIDE.md` 提供完整步骤：
   - 克隆仓库和软件版本；
   - Docker 推荐路线；
   - 原生基础设施路线；
   - IDEA 启动八个 Java 进程；
   - Vue 前端安装与启动；
   - 冒烟测试；
   - Chrome 扩展安装与手动导入；
   - 常见错误和端口表；
   - 发给同学前的检查清单。

## 约束

- 不修改课程依赖版本。
- 不增加新的微服务或安装器。
- 不承诺自动登录、自动翻页或绕过 BOSS 安全验证。
- 所有命令以仓库根目录下的 PowerShell 为执行位置。
- 不依赖开发者本机 `Administrator` 用户目录。

## 验证

- 检查文档中的本地文件、脚本和启动类均真实存在。
- 检查 README 能链接到详细指南。
- 执行 Markdown 命令/路径静态核对，并重新读取远端 README 验证公开发布结果。
